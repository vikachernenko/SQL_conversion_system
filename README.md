# 🏛 Universal AI SQL Assistant

## 📝 Описание проекта

Это универсальная BI-система, которая позволяет загружать базы данных и задавать к ним вопросы на естественном языке. Система автоматически преобразует вопрос в SQL-запрос, выполняет его и визуализирует результат.

**Особенности:**

- **Поддержка языков:** Русский, English, Татарча.
- **Поддержка СУБД:** SQLite, PostgreSQL.
- **Интеллектуальный RAG:** Возможность выбора источника знаний между мировыми датасетами **Spider** и **BIRD** для улучшения точности генерации.

---

## 🏗 Архитектура и компоненты

Программа реализована на языке Python и состоит из следующих логических блоков:

1.  **Интерфейс и управление состоянием (Streamlit):**
    - Боковая панель для выбора настроек (язык, диалект, источник RAG).
    - Система `st.session_state` для хранения пути к загруженной базе данных.
2.  **Обработка данных (DB Engine):**

    - **Сценарий `.db/.sqlite`:** Прямое подключение к файлу.
    - **Сценарий `.sql`:** Создание локальной базы данных SQLite и выполнение загруженного скрипта для воссоздания структуры и данных «на лету».

3.  **База знаний (Knowledge Base - RAG):**

    - **Класс SpiderKnowledgeBase:** Стриминговая загрузка датасета `xlangai/spider` для получения примеров «Вопрос-SQL».
    - **Класс ProKnowledgeBase:** Стриминговая загрузка датасета `birdsql/bird_mini_dev` (сплит `mini_dev_sqlite`) для получения сложных бизнес-примеров с контекстом (Evidence).

4.  **Контекстный анализатор (`get_db_context`):**

    - Автоматическая инспекция загруженной базы: извлечение имен таблиц, колонок, типов данных и внешних связей (Foreign Keys).
    - Объединение схемы БД с примерами из выбранного RAG-источника.

5.  **Генератор SQL с самокоррекцией (`ask_ai_for_sql`):**

    - Использование локальной модели **Llama 3** через Ollama.
    - **Цикл Self-Correction:** В основном цикле программы заложено 3 попытки выполнения запроса. Если база возвращает ошибку, текст ошибки передается обратно в ИИ для генерации исправленного кода.

6.  **Визуализация (Plotly):**
    - Автоматическое определение числовых колонок в результате.
    - Рендеринг столбчатой диаграммы (`px.bar`), если в данных есть числа.

---

## 🔄 User Flow (Путь пользователя)

1.  **Конфигурация:** В боковой панели пользователь выбирает язык, СУБД и источник знаний (Spider или BIRD).
2.  **Загрузка данных:** Пользователь загружает файл `.db` или `.sql`. Если загружен `.sql`, система автоматически создает базу.
3.  **Просмотр схемы:** В раскрывающемся списке можно выбрать таблицу и увидеть первые 5 строк для ознакомления.
4.  **Запрос:** Пользователь вводит вопрос (например, на татарском языке).
5.  **AI-обработка:**
    - Система собирает контекст (Схема БД + Примеры из RAG).
    - ИИ генерирует SQL-код.
    - В случае ошибки выполнения система делает до 2-х дополнительных попыток исправления кода.
6.  **Результат:** Выводится итоговый SQL-код, таблица с данными и интерактивный график.

## 🛠 Запуск

### 1. Требования

- Python 3.10+
- Установленное приложение [Ollama](https://ollama.com/)

### 2. Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/vikachernenko/SQL_conversion_system.git
```

Перейдите в рабочую директорию проекта:

```bash
cd SQL_conversion_system
```

Создайте виртуальное окружение:

```bash
python -m venv venv
```

Активируйте его:

```bash
windows: \venv\Scripts\activate'

unix: source "/venv./bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

p.s. при необходимости после загрузки завистимостей перезагрузите проект, только не забудьте заново активировать виртуальное окружение

### 3. Подготовка модели

Запустите Ollama и скачайте модель Llama 3:

```bash
ollama pull llama3
```

### 4. Запуск программы

```bash
streamlit run app.py
```
